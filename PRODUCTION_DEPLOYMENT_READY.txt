🚀 CHECKLIST PRE-DEPLOYMENT

PASO 1: Preparación Local
  [ ] Clonar/descargar proyecto
  [ ] Generar JWT_SECRET: python -c "import secrets; print(secrets.token_urlsafe(32))"
  [ ] Crear cuenta MongoDB Atlas
  [ ] Obtener connection string MongoDB
  [ ] Tener dominio registrado y disponible

PASO 2: Archivos de Configuración
  [ ] Copiar backend/.env.production.template → backend/.env
  [ ] Llenar MONGO_URL con tu connection string
  [ ] Llenar JWT_SECRET con valor generado
  [ ] Configurar CORS_ORIGINS con tu dominio
  [ ] Llenar API_URL y API_KEY
  [ ] Copiar frontend/.env.production → frontend/.env
  [ ] Configurar REACT_APP_BACKEND_URL

PASO 3: Infraestructura Cloud
  [ ] Crear cuenta en proveedor (DigitalOcean/AWS/Azure)
  [ ] Crear máquina virtual/Droplet
  [ ] Configurar SSH keys
  [ ] Anotar IP de servidor
  [ ] Configurar firewall/security groups

PASO 4: DNS y Dominio
  [ ] Ir al registrador del dominio
  [ ] Crear A record apuntando a IP del servidor
  [ ] Esperar propagación DNS (5-30 minutos)
  [ ] Verificar: nslookup yourdomain.com

PASO 5: Instalación en Servidor
  [ ] SSH al servidor
  [ ] Instalar Docker y Docker Compose
  [ ] Instalar Nginx
  [ ] Instalar Certbot
  [ ] Clonar proyecto
  [ ] Colocar archivos .env

PASO 6: Configuración Nginx + SSL
  [ ] Copiar nginx.conf a /etc/nginx/sites-available/coner
  [ ] Editar dominio en nginx.conf
  [ ] Habilitar sitio
  [ ] Verificar: sudo nginx -t
  [ ] Obtener certificado Let's Encrypt
  [ ] Recargar Nginx

PASO 7: Deployment
  [ ] docker-compose build
  [ ] docker-compose up -d
  [ ] Verificar: docker-compose ps
  [ ] Revisar logs: docker-compose logs -f

PASO 8: Verificación
  [ ] curl https://yourdomain.com (frontend)
  [ ] curl https://yourdomain.com/api/health (backend health)
  [ ] curl https://yourdomain.com/docs (API documentation)
  [ ] Acceder desde navegador
  [ ] Probar flujo completo de usuario

⚠️ IMPORTANTE - SEGURIDAD EN PRODUCCIÓN

ANTES DE DEPLOYEAR:

✓ JWT_SECRET:
  • Generar nuevo (no usar el de template)
  • Mínimo 32 caracteres
  • Completamente aleatorio
  • Nunca commitir a Git

✓ API_KEY:
  • Usar credenciales específicas para producción
  • Cambiar de las de desarrollo
  • Rotar cada 90 días

✓ Credenciales MongoDB:
  • Usuario específico para producción
  • Contraseña fuerte (>16 carácteres)
  • IP whitelist en MongoDB Atlas

✓ Backups:
  • MongoDB: Automático en Atlas (diario)
  • App: Snapshots periódicos del servidor

✓ SSL/HTTPS:
  • Obligatorio en producción
  • Let's Encrypt (GRATIS, automático)
  • Auto-renewal cada 3 meses

✓ Firewall:
  • Puerto 22 (SSH): Solo tu IP
  • Puerto 80/443 (HTTP/HTTPS): 0.0.0.0/0
  • Puerto 8000 (Backend): Cerrado (solo Nginx)
